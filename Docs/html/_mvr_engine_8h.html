Function Documentation

BOOL MvrMiniIsAvailable (  ) 

Checks whether Mover Mini is currently available; it returns YES if it is, or NO if it is not.

If this function returns NO, no Mover Mini function, method, constant or class is safe for use. Call this function before using any of the above. Caching the result of this function call is allowed (that is, the value will not change during the lifetime of the application).

Variable Documentation

NSString* const kMvrEngineDidFindEndpointNotification

This notification is posted whenever the engine finds a new endpoint, that is, a remote device running a compatible version of Mover Mini or Mover.

The new endpoint is the object at the kMvrEngineEndpointKey key in the notification's info dictionary.

Please note that an endpoint may be found multiple times by the engine. It's up to you to filter multiple appearances of the same endpoint in your code. (The MvrTable automatically does this for its own part when displaying them as arrows on the table.)

NSString* const kMvrEngineDidFinishReceivingNotification

This notification is posted whenever an engine has successfully received an allowed item on your behalf.

The transfer has finished for the incoming transfer object at the user info's kMvrEngineIncomingKey. If the transfer completed successfully, the received MvrMiniItem is found at the user info's kMvrEngineItemKey; otherwise, this key will be unset.

The notification is sent before the MvrEngine::knownItems set is altered.

NSString* const kMvrEngineDidFinishSendingNotification

This notification is posted whenever the engine has finished sending an item on your behalf.

The item that was sent is available in the user info dictionary under the kMvrEngineItemKey. A MvrMiniOutgoing object describing the transfer is available under the kMvrEngineOutgoingKey. If an error occurred while sending, it's available in the user info dictionary under the kMvrEngineErrorKey.

NSString* const kMvrEngineDidLoseEndpointNotification

This notification is posted whenever the engine finds that an endpoint is no longer available.

The disappeared endpoint is the object at the kMvrEngineEndpointKey key in the notification's info dictionary.

NSString* const kMvrEngineDidStartReceivingNotification

This notification is posted whenever the engine begins receiving an item on your behalf.

If an item is sent by another client whose type is not in MvrEngine::allowedItemTypes, the item is silently rejected; this notification is not sent in this case.

The information about the incoming transfer is available through the object at the user info's kMvrEngineIncomingKey.

NSString* const kMvrEngineDidStartSendingNotification

This notification is posted whenever an engine has started sending an item on your behalf.

This happens whenever the user slides an item off the table to be sent.

The item to be sent is available in the user info dictionary under the kMvrEngineItemKey. The endpoint is available under the kMvrEngineEndpointKey. A MvrMiniOutgoing object describing the transfer is available under the kMvrEngineOutgoingKey.

NSString* const kMvrEngineEndpointKey

The object for this key in a kMvrEngineDidFindEndpointNotification or kMvrEngineDidLoseEndpointNotification's user info dictionary is an endpoint object (conforming to the MvrEndpoint protocol).

NSString* const kMvrEngineErrorKey

The object for this key in a kMvrEngineDidFinishSendingNotification is the error that prevented the item from being sent.

If no error occurred, this key is absent from the user info dictionary.

NSString* const kMvrEngineIncomingKey

The object for this key in a kMvrEngineDidStartReceivingNotification or kMvrEngineDidFinishReceivingNotification is a MvrMiniIncoming object describing an incoming transfer.

NSString* const kMvrEngineItemKey

The object for this key in a kMvrEngineDidFinishReceivingNotification, kMvrEngineDidStartSendingNotification or kMvrEngineDidFinishSendingNotification is a MvrMiniItem representing the latest successfully received item.

NSString* const kMvrEngineOutgoingKey

The object for this key in a kMvrEngineDidStartSendingNotification or kMvrEngineDidFinishReceivingNotification is a MvrMiniOutgoing object describing an outgoing transfer.
