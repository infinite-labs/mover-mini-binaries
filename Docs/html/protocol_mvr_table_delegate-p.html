Detailed Description

A table delegate will be informed of events coming from a MvrTable instance.

See the MvrTable::delegate property for more information.

- (BOOL) moverTable: (MvrTable *)  table
shouldShowForNewlyAddedItem: (MvrMiniItem *)  item
withCause: (MvrTableShowCause)  cause 
[optional]

This method is called every time the table desires to display itself.

This can happen due to various causes, both programmatically (as the result of your code calling the showByAddingItem: (MvrTable) method) or because a new item has been received and the table desires to show it.

You can return NO to prevent displaying the table for the item. This method is not called for all received items; the table may not bother calling it if it determines it cannot show itself anyway.

- (void) moverTableDidChangeCanShow: (MvrTable *)  table   [optional]

Called when the screen configuration has changed and the table has potentially changed its MvrTable::canShowTable property.

Call that method to retrieve the new value of the property.

Please note that the property might not have changed since the last time this delegate method was called.
